# Swans ShuttleHub - AI Build Specification

## Project Overview

Build a production-quality Progressive Web App (PWA) called **Swans ShuttleHub** for Swans Travel.

The application is for recording passenger loading figures on an internal staff shuttle service and providing a live operational dashboard for the traffic office.

The application must work perfectly on iPhone, Android, tablets and desktop browsers.

It must be installable as a Progressive Web App (PWA) and have a modern, fast, mobile-first design.

The entire application should be built with scalability in mind so future operational modules can be added.

---

## Technology Stack

Use:

- React
- TypeScript
- Tailwind CSS
- Supabase (Authentication + Database + Realtime)
- React Router
- Google Maps (or Mapbox if preferred)
- Vercel compatible deployment
- PWA support
- Responsive design

Use clean, reusable components and production-ready code.

# DRIVER APPLICATION

Driver names are managed from the Admin Portal.

Automatically record the driver's name against every submission.

## GPS Tracking

When the duty starts:

Request permission to use the phone's GPS.

If permission is granted:

Begin recording GPS position every 30 seconds.

Capture:

• Latitude
• Longitude
• Timestamp
• Driver
• Duty
• Speed (if available)
• Heading (if available)

If offline, store updates locally and automatically upload them once internet returns.

---

## Geofencing

Create configurable geofences around shuttle stops.

Example stops:

Royal Well Bus Station

Benhall

Arle Court

When the phone enters a stop geofence:

Automatically display the scheduled departure for that stop.

The driver should never need to manually select a departure if GPS determines the correct stop.

---

## Passenger Recording

The only information the driver enters is:

Passenger Count

Use large + and - buttons with a numeric display.

Example:

Passenger Count

[-] 12 [+]

Large Submit button

When submitted automatically record:

Date

Driver

Duty

Scheduled Departure Time

Actual Submission Time

Stop Name

Latitude

Longitude

Passenger Count

Unique Record ID

After saving automatically move to the next scheduled departure.

Each departure may only be submitted once.

If already submitted display:

"This departure has already been recorded."

---

## Offline Support

If internet is unavailable:

Store all submissions locally.

Automatically upload when internet connection returns.

No records should ever be lost.
